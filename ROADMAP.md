# TakeoffLens roadmap

The roadmap describes direction, not delivery dates. A discipline becomes
active only after its rules, review workflow, and benchmark evidence are ready.

## v0.2 - Accuracy and product foundation

- [x] TakeoffLens product and Plugin identity
- [x] active and planned discipline catalog
- [x] Electrical and ELV symbol taxonomy
- [x] candidate pool and filter-stage audit trail
- [x] source, template, and candidate provenance
- [x] reviewed precision, recall, F1, and miss attribution
- [x] local Codex Plugin packaging and migration path
- [x] public installation, architecture, and development documentation

## v0.2.x - Electrical and ELV hardening

- [ ] expand reviewed benchmarks across multiple projects and drawing standards
- [ ] improve project legend onboarding and template management
- [ ] add revision-comparison evidence and reporting contracts
- [ ] refine output formats for BOQ and downstream estimating workflows
- [ ] add certified Hermes configuration and compatibility tests
- [ ] improve install diagnostics and managed-runtime recovery

## v0.3 - Portable MCP product

- [ ] separate transport from the Core service boundary
- [ ] add remote Streamable HTTP MCP
- [ ] add authentication and project-scoped storage options
- [ ] build a ChatGPT App adapter over the shared MCP tools
- [ ] retain local-first deployment for confidential drawings
- [ ] define a stable adapter contract for other MCP hosts

## Future Discipline Packs

- [ ] Mechanical and HVAC
- [ ] Plumbing and Sanitary
- [ ] Fire Protection
- [ ] Architectural
- [ ] Structural
- [ ] specialist packs such as BMS, Solar, Medical Gas, and Landscape

## Raster and document intelligence

- [ ] benchmarked OCR for notes, tags, and legend text
- [ ] tiled high-resolution raster inference
- [ ] reviewed object-detection pipeline for scanned drawings
- [ ] document set classification and sheet relationship analysis
- [ ] revision and cross-sheet consistency checks

TakeoffLens will not claim universal PDF understanding. Each capability should
be supported by explicit scope, evidence, and a review contract.
